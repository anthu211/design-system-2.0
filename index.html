Characteristics
  • Best for comparing values across discrete time periods or categories
  • Supports grouped bars (multiple series per x-axis point)
  • Recommended max: 12 groups, 4 series per group
  • Y-axis starts at 0 unless data range justifies otherwise
Behavior
  • Hover over any bar to dim to 78% opacity (focus effect)
  • Each bar uses a role-based color token (Critical, High, Medium, Low)
  • Title attribute shows exact value on native tooltip
  • Responsive: bars shrink proportionally with container
Usage Guidelines
  • Use when comparing magnitude across time (monthly, quarterly)
  • Avoid for part-to-whole analysis — use donut instead
  • Keep color count ≤ 5 for readability
  • Always include a legend when using multiple series

Characteristics
  • Best for showing trends over continuous time (daily, monthly)
  • Minimum 3 data points required; optimal 6–12
  • Gradient fill under line adds visual weight to area
  • Data points marked with filled circles for exact value reference
Behavior
  • SVG polyline rendered via buildLineChart()
  • Fully responsive — recalculates on container width change
  • X and Y axis labels auto-generated from data arrays
  • Re-renders when switching to/from charts page
Usage Guidelines
  • Use for time-series data only, not discrete categories
  • Multi-line variant: add second polyline with different stroke color
  • Avoid more than 3 lines — use small multiples instead
  • Always label the x-axis with time units
